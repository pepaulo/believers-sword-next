# Default Databases

Seed/content databases shipped with the app and consumed by the desktop client
(see `Believers Sword App`). Each *content* DB has a sibling version file in the
form `{"version": N}`. The client compares that number against the copy it has
installed and replaces its local DB when the shipped version is higher — so
**whenever you ship an updated DB you must bump its version file**, otherwise
clients keep the old data.

## Databases

| File                     | Version file                   | Description                                                              |
| ------------------------ | ------------------------------ | ------------------------------------------------------------------------ |
| `cross_references.db`    | `cross_references_version.json`| Cross-reference links between verses.                                    |
| `strongs.db`             | `strongs.version.json`         | Greek + Hebrew Strong's number dictionary (table `strongs_dictionary`).  |
| `Dictionary.db`          | —                              | Bible dictionary entries. (No version file — shipped as-is.)             |
| `Store.db`               | —                              | Default app store / settings seed.                                       |
| `devotional/devotionals.db` | `devotional/version.json`   | Daily devotional content.                                                |
| `games/fp/game_seed.db`  | `games/fp/version.json`        | "Fill in the blank" (fp) game seed data + `images/`.                     |
| `games/qa/game_seed.db`  | `games/qa/version.json`        | "Question & answer" (qa) game seed data.                                 |
| `games/tf/game_seed.db`  | `games/tf/version.json`        | "True / false" (tf) game seed data.                                      |

## Version files

Each version file is a small JSON document:

```json
{
    "version": 3
}
```

Bump `version` by 1 each time the matching `.db` is regenerated and shipped.
The version number — not the DB file's contents or timestamp — is what triggers
the client to replace its local copy.

## Strong's dictionary (`strongs.db`)

Generated by `scripts/Generate Strong Number Sqlite/generate_strongs_db.py` from
the Greek and Hebrew JSON sources. To ship an update: regenerate the DB, copy it
here as `strongs.db`, and bump `strongs.version.json`. See
`scripts/Readme.md` for the generator details and the `strongs_dictionary`
schema.
